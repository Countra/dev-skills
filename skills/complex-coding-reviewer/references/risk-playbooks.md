# Risk Playbooks

只读取与当前目标相关的章节。未命中的领域无需逐项报告。

## 安全与权限

- 身份验证、授权检查和租户/资源所有权是否在可信边界执行
- 路径、命令、模板、查询和反序列化输入是否可注入
- 秘密、token、日志和错误信息是否越界暴露
- 默认权限、危险操作确认和失败回滚是否安全

高风险但证据不足时请求安全专业审查，不把普通 clean review 当作安全认证。

## 隐私与数据治理

- 收集、存储、日志、导出和删除的数据是否符合批准目的
- 敏感字段是否最小化并受访问控制
- 保留期限、撤回、匿名化和跨边界传输是否被考虑

## 并发与数据完整性

- 共享状态是否存在竞态、丢失更新、重复执行或顺序假设
- 事务边界、幂等键、重试和崩溃恢复是否一致
- 部分失败能否留下不可恢复或误报成功的状态

## 性能与资源

- 复杂度、查询数量、内存、文件句柄和进程生命周期是否随输入失控
- 超时、取消、背压、并发上限和清理是否真实生效
- 性能 finding 必须给出可触发规模或证据，不凭感觉推测

## API 与数据兼容

- 公共字段、状态码、默认值、序列化和错误语义是否改变
- 数据迁移是否兼容旧数据、回滚和混合版本
- 调用方、文档、生成代码和测试是否同步

## UI、可访问性与国际化

- 键盘、焦点、语义标签、对比度和错误反馈是否可用
- 动态内容是否造成布局覆盖、状态丢失或不可恢复操作
- 文案、日期、数字、复数、编码和 RTL 是否与项目要求一致

## 删除与依赖

- 删除的模块、脚本、配置、资源和文档是否仍被引用
- 替换依赖是否同步 manifest、lockfile、构建、部署和许可证
- 删除是否破坏恢复、安装、CI 或跨平台路径
