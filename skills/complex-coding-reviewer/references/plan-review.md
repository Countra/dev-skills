# Plan Review

## 目标与范围

- 目标和完成标准是否可观察、可验证
- 包含与不包含范围是否清晰
- 关键约束、用户决策和授权是否被保留
- 是否出现未获批准的功能、依赖或外部写入

## 事实与决策

- 关键仓库事实是否来自当前代码和配置
- 时效信息、平台行为和依赖选择是否有适用的一手来源
- 方案选择是否符合项目现状，并说明主要权衡和退出路径
- 不确定项是否已解决、明确阻塞，或被错误当作事实

## 阶段与可实施性

- 阶段边界是否对应可交付结果，而不是任意文件分组
- 依赖方向是否正确，是否存在循环、隐藏前置条件或不可恢复中间态
- 每阶段范围、验证和风险是否足以让 Executor 实施
- 失败、回滚、迁移和跨平台路径是否与风险匹配

## 验证与审查

- 每个关键行为是否有真实、可运行或可观察的验证
- 验证是否覆盖错误路径、共享行为和兼容边界
- 高风险阶段和最终集成是否要求 independent review
- 未能验证的内容是否明确写为 gap，而不是假设通过

plan-review 不要求固定章节、稳定需求矩阵或文档长度。只报告会影响正确实施、验证、风险或批准边界的问题。
