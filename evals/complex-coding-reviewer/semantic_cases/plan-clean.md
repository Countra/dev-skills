# 规划样本：完整的配置校验方案

目标：为配置文件新增严格 schema 校验；未知字段返回 `CFG_UNKNOWN_FIELD`，字段类型错误返回 `CFG_INVALID_TYPE`。

范围：只修改 parser、错误码和对应测试，不迁移存量配置，不改变合法配置的解析结果。

阶段：

1. 在 parser 边界接入现有 schema validator，并保持错误码映射集中在 `config_errors.py`。
2. 增加合法配置、未知字段、类型错误、空对象和嵌套错误路径的单元测试。
3. 运行配置模块完整测试和公共 CLI 回归；失败时回退本阶段变更，不修改存量数据。

验收：测试命令及预期退出码已固定，非法输入断言错误码和字段路径，合法输入断言结果与基线一致。

审查请求：使用 `plan-review` 判断方案是否完整、可实施且不过度扩展范围。
